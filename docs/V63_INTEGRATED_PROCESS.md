# Reverie v63 통합 공정서

최종 갱신: 2026-03-10

## 1. 목적

이 공정서는 Reverie를 안정적으로 개선하기 위한 작업 순서, 실패 시 재시작 규칙, 검증 절차를 정의한다.

## 2. 공정 원칙

1. 구조 수정 전 현재 코드 기준 사실을 먼저 확인한다.
2. 팩 규칙을 깨는 기능 추가는 금지한다.
3. 대형 리팩토링은 단계별로 쪼개고, 각 단계마다 테스트와 문서 갱신을 끝낸다.
4. 실패 시 복구 가능한 중간 산출물을 남긴다.

## 3. 단계별 공정

### Phase 0. 안정화

목표:

- 역방향 import 제거
- 체크포인트 재개를 GUI 경로에 연결
- 비GUI 테스트 확보

완료 기준:

- `runtime_utils` 공용화 완료
- GUI 재개 옵션 전달 완료
- mock 기반 GUI-safe 테스트 통과

### Phase 1. 구조 분해

목표:

- `scenario_planner`, `script_writers`, `orchestrator`, `image_pipeline`, `tts_manager` 분해

작업 순서:

1. public API 목록 동결
2. 내부 메서드 묶음별 하위 서비스 추출
3. import 순환 여부 확인
4. 기존 테스트 + 회귀 테스트 실행

산출물:

- 얇은 orchestrator
- 단계 서비스별 파일
- 이전 파일의 호환 래퍼 또는 명확한 마이그레이션 문서

### Phase 2. 신뢰성 강화

목표:

- 예외 삼키기 제거
- 로그 표준화
- last failed job 재시도 지원

작업 순서:

1. `except Exception/pass` 목록화
2. 사용자 오류 / 외부 엔진 오류 / 내부 버그를 분리
3. 단계별 실패 태그 강제
4. 최근 실패 작업 registry 추가

### Phase 3. 성능 최적화

목표:

- 생성 병목 축소
- temp 및 cache 정책 정리

작업 순서:

1. 단계별 소요 시간 수집
2. TTS 캐시와 이미지 캐시 우선 적용
3. 렌더 temp 정리 자동화
4. 워커 상주화 또는 provider 분리

### Phase 4. 경쟁력 기능

목표:

- 팩 검증기
- 트렌드 기반 기획 후보군
- 성과/비용 대시보드
- 하이브리드 provider 라우팅

## 4. 실패 시 재시작 규칙

### 개발 작업 실패

- 리팩토링 중 실패하면 직전 통과 테스트 지점부터 재개한다.
- 큰 파일은 한 번에 전부 바꾸지 말고, 서비스 단위 추출 후 테스트를 끊는다.

### 생성 작업 실패

- 생성 파이프라인은 단계 완료 시 체크포인트를 저장한다.
- 재시작 시 동일 JSON과 동일 프로젝트명을 기준으로 이어간다.
- 성공 후 체크포인트는 삭제한다.

### 배치 작업 실패

- 개별 작업은 실패 상태를 남기되, 나머지 큐는 계속 가능해야 한다.
- 향후 `retry failed job` 기능을 추가해 마지막 실패 항목을 그대로 재개한다.

## 5. 검증 공정

### 자동 검증

필수:

- `pytest` 기반 import/runtime 테스트
- 체크포인트 재개 옵션 전달 테스트
- 핵심 리팩토링 회귀 테스트
- `compileall` 또는 동등 수준 정적 검증

권장:

- 대용량 파일 import smoke test
- 배치 큐 atomic write 테스트

### 수동 검증

사용자는 GUI를 직접 열고 아래를 확인한다.

1. 체크포인트 재개 체크박스 노출
2. 직접 생성 경로 정상 실행
3. 미리보기 승인 후 생성 경로 정상 실행
4. 배치 큐 저장 후 재실행 시 옵션 유지
5. 실패 후 동일 프로젝트 재실행 시 이전 단계 스킵

## 6. 문서 공정

문서 업데이트 순서:

1. 설계 변경 시 `V63_INTEGRATED_DESIGN.md`
2. 요구사항 변경 시 `V63_INTEGRATED_SPEC.md`
3. 절차 변경 시 `V63_INTEGRATED_PROCESS.md`
4. 그 외 문서는 필요 시 링크만 업데이트

중복 처리 규칙:

- 중복 문서는 삭제보다 먼저 `superseded` 안내로 축약
- 역사 문서는 새 정본을 가리키는 용도로만 유지
