## 0. 핵심 원칙

**레베리는 팩-클라이언트 구조다.**
- 엔진(프로그램) = 월구독 → 실행 권한
- 팩(.revpack) = 일회성 → 장르 권한
- 라이선스 타입(A/H/T/M)이 접근 가능한 팩을 결정

### 3-2. 팩 배포 구조 대입

**현재 팩 로딩 흐름:**
```
load_pack_by_id(pack_id)
  → .revpack 파일 존재? → load_pack() → Fernet 복호화 → ACTIVE_PACK 갱신
  → 없으면 → 디렉토리 팩 → _load_pack_from_json()
```

**팩 암호화 방식 (기존):**
```python
# src/config/pack_config.py L63-65
PACK_ENCRYPTION_SALT = b'ReveriePack2024Salt!'
PACK_ENCRYPTION_PASSWORD = b'ReverieStudio_PackEncryption_v57'
# Fernet 방식 — 헤더 'gAAAAA'로 시작하면 자동 복호화
```

**배포 방식 설계:**
```
판매 완료
  → 구매자에게 라이선스 키 발급 (Firebase)
  → 라이선스 키에 owned_packs 필드에 팩 ID 추가
  → 레베리 실행 시 Firebase 조회 → owned_packs 확인
  → 보유한 팩만 UI에 표시
  → .revpack 파일은 Firebase Storage에서 자동 다운로드 (향후)
  → 현재는: .revpack 파일 직접 이메일/링크 전달 (초기 운영)
```
